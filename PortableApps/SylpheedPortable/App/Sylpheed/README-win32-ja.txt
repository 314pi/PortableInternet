説明
====

Sylpheed はメールソフト(電子メールクライアント、メーラ、 MUA)です。
シンプル、軽量かつ高機能で使いやすいことが特徴です。

POP3、IMAP4rev1、NNTP など多様なプロトコルに対応しており、SSLにも対応して
います。

また、Sylpheed は国際化と多言語表示に対応しており、現在29ヶ国語以上に翻訳
されています。日本語の処理は、文字コードの自動判別(ISO-2022-JP/EUC-JP/
Shift_JIS/UTF-8)、機種依存文字対応など、高いレベルで対応しています。

強力なフィルタリング機能や検索機能、外部プログラムを利用した学習型迷惑メール
対策機能も装備しています。

また、 GnuPG がインストールされている場合は PGP 機能が利用できます。

インストール
============

(インストーラ版の場合)
インストーラの exe ファイルを実行してください。インストーラが起動したら
その指示に従ってください。

(zip アーカイブ版の場合)
zip アーカイブを適当な場所に展開してください。
その後、 sylpheed.exe を実行してください。


アップデート
============

(インストーラ版の場合)
Sylpheed が起動していない状態でインストーラを起動してください。
自動的に以前のバージョンのアンインストールを行った後、インストールが
開始されます。

(zip アーカイブ版の場合)
Sylpheed が起動していない状態で、 zip アーカイブの中身を以前のバージョンが
インストールされたフォルダに上書きコピーしてください。
メジャーバージョンアップの場合などは不要なファイルが残る場合もあるため、
以前のバージョンを削除してからコピーすることを推奨します。

また、デフォルトの設定では、 Sylpheed の起動時に新しいバージョンがリリース
されているかどうかをサーバに問い合わせ、存在する場合はダイアログが表示されます。


使用方法
========

デスクトップの Sylpheed アイコン、または「スタートメニュー - Sylpheed -
Sylpheed」から Sylpheed を起動してください。
使い方の詳細は「スタートメニュー - Sylpheed - Sylpheed Manual」または
「ヘルプ - マニュアル」を参照してください。


設定ファイル
============

設定ファイルやメールボックスは通常は以下の場所(%APPDATA%\Sylpheed)
に保存されます。

C:\Users\(ユーザ名)\AppData\Roaming\Sylpheed

設定ファイルの場所を変更したい場合は、コマンドラインオプション
「--configdir "ディレクトリ名"」で指定することができます。
sylpheed.exe のショートカットを作成し、「リンク先」に以下のように追加
してください。

"C:\Program Files\Sylpheed\sylpheed.exe" --configdir "D:\Sylpheed"

また、 sylpheed.exe と同じ場所に sylpheed.ini ファイルがあればそれを
起動時に読み込みます。これを用いることで、ドライブレター等に依存しない
完全にポータブルな Sylpheed にすることができるので、 USB メモリなどに
インストールして使用することが可能です。
(注: インストーラ版、また GnuPG と GPGME はレジストリを使用するので
注意してください)

サンプルの INI ファイルが sample-sylpheed.ini という名前で同梱されて
います。必要に応じてコピー・リネームして使用してください。
ipcport はプロセス間通信に使用するソケットのポート番号を指定します
(既定値は50215番です)。
configdir は設定ファイルの場所を sylpheed.exe のあるフォルダからの相対
パス、または絶対パスで指定します。


迷惑メール対策機能
==================

迷惑メール対策機能を有効にするには、別途
SylFilter ( http://sylpheed.sraoss.jp/sylfilter/ ) もしくは
bsfilter ( http://bsfilter.org/ ) をインストールする必要があります。
簡単なインストール方法としては、 sylfilter.exe もしくは bsfilterw.exe を
sylpheed.exe と同じディレクトリにコピーすれば OK です。

Sylpheed のインストーラには SylFilter と bsfilter が同梱されているので、
設定を有効にすればそのまま利用できます。性能と検出精度の点から、 SylFilter
の利用をおすすめします。

SylFilter
---------

SylFilter の注意点は特にありません。
SylFilter の設定、データベースは以下の場所に保存されます。

C:\Users\(ユーザ名)\AppData\Roaming\SylFilter

bsfilter
--------

以前別の場所に bsfilter をインストールしていた場合はデータベースファイルの
場所が変わるので注意してください。

ただし、Sylpheed を実行するユーザに管理者権限がない場合は、bsfilter が
ファイルを書き込めずにエラーになります。その場合以下のいずれかの方法を
行ってください。

* 管理者権限をもつユーザで実行する
* 環境変数 BSFILTERHOME に書き込み可能なディレクトリを指定する
  - 「コントロールパネル - システムとメンテナンス - システム -
     システムの詳細設定 - 環境変数 - (ユーザ名) のユーザー環境変数」
  - 「変数名」に BSFILTERHOME、「変数値」に
    「C:\Users\(ユーザ名)\AppData\Roaming\Sylpheed」を入力して「OK」を押す
* 「全般の設定 - 迷惑メール - 学習コマンド」の各コマンドに
  --homedir "ディレクトリ" を追加する
  例) bsfilterw.exe --homedir (次の行に続く)
       "C:\Users\(ユーザ名)\AppData\Roaming\Sylpheed" -C -s -u

なお、7/8/10の場合、標準の Program Files フォルダにインストールした場合に限り、
自動的に「C:\Users\(ユーザ名)\AppData\Local\VirtualStore\Program Files\Sylpheed」
以下にリダイレクトされるため、上記の対策は特に必要ありません。

Sylpheed で動作確認済みのものを以下の場所にも置いています。

  http://sylpheed.sraoss.jp/sylpheed/others/bsfilter-1.0.17.rc4.tgz
  http://sylpheed.sraoss.jp/sylpheed/others/bsfilter-1.0.17.rc4/
    bsfilter/bsfilterw.exe

注: bsfilter-1.0.16.tgz に同梱されている bsfilterw.exe は動作に問題があります。
    また、過去のバージョンには一部のメールで無限ループ状態に陥るバグがあります。
    bsfilter-1.0.17.rc4 以降をお使いください。


PGP 機能
========

PGP 機能を有効にするには、 GnuPG
( http://www.gnupg.org/(en)/download/index.html#auto-ref-1 )
をインストールする必要があります。


mailto リンクと .eml ファイルの関連付け
=======================================

zip アーカイブ版からインストールした場合に Sylpheed を mailto: プロトコルの
ハンドラとして登録し、また拡張子 .eml と関連付けたい場合は、同梱の
sylpheed-mailto-protocol_admin.reg ファイルと sylpheed-mailto-protocol_user.reg
ファイルをダブルクリックし、レジストリにエントリを追加してください。
インストーラ版の場合はインストール時に自動的に登録できます。

また、他のメールソフトをデフォルトとして指定した後、 Sylpheed をデフォルトに
戻したい場合も同様にレジストリにエントリを追加してください。

C:\Program Files\Sylpheed\sylpheed.exe 以外の場所にインストールした場合は、
パスを適宜書き換えてください。


アンインストール
================

(インストーラ版の場合)
コントロールパネルの「プログラムの追加と削除」から、 Sylpheed を選択して
「変更と削除」をクリックしてください。

(注: インストールされているバージョンが2.2.3以前の場合、インストールフォルダ
中のファイルはすべて削除されてしまうため、インストールフォルダ以下にメール
ボックスを作成している場合や、 bsfilter 等を使用している場合は注意してくだ
さい。2.2.4以降からはインストーラによって作成されたファイル・フォルダ以外は
削除しません)

(zip アーカイブ版の場合)
展開したフォルダごと削除してください。

設定ファイルやメールボックスは自動では削除されません。不要な場合は、手動で
削除してください。


SSL 証明書の追加方法
====================

Sylpheed は OpenSSL ライブラリを使用して SSL による通信を行っています。
そのため、証明書の検証に Windows の証明書ストアを直接使用できません。
Sylpheed に証明書を追加するには以下の方法で行います。

1. CA証明書ファイルを開き、 Base 64 encoded X509 (.CER) 形式でエクスポート
   します。
2. Sylpheed をインストールしたフォルダ以下の etc\ssl\certs\certs.crt
   ファイルを改行コードが LF のものに対応したテキストエディタで開きます。
3.  1. でエクスポートしたファイルを同様に開き、 2. で開いたファイルの
   先頭または末尾にコピー&貼り付けで追加します。
4. 編集したファイルを 2. の certs.crt に上書き保存します。
   ユーザ毎に管理したい場合は
   C:\Users\(ユーザ名)\AppData\Roaming\Sylpheed\certs.crt
   に保存してください。

なお、証明書の検証に失敗した場合に「常に受け入れる」を選択すると、
証明書がローカルに保存され、次回以降自動的に受け入れるようになります。


その他の注意点
==============

現在のバージョンでは、 Unix 版に実装されているアクション機能など一部の機能が
実装されていません。

Sylpheed の Windows 版はリモート操作のためのプロセス間通信を TCP socket
を使用して実装しています。そのため、ループバックアドレス(127.0.0.1)の
50215番ポートを消費します。

バグ報告の際は「ツール/ログウィンドウ」、または設定ファイルフォルダ以下の
sylpheed.log の内容なども提供してください。
